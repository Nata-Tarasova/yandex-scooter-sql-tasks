# Тестирование базы данных СУБД PostgreSQL (Яндекс Самокат) 🛴

Проект содержит решения практических задач по валидации данных на бэкенде сервиса Яндекс Самокат с использованием SQL-запросов.

## 🛠️ Используемые инструменты
* **СУБД:** PostgreSQL
* **Инструмент администрирования:** DBeaver (v26.0.5)
* **Тестирование API:** Postman
* **Клиентская часть:** Мобильное приложение курьера (тестовый стенд)

---

## 📋 Выполненные задачи

### 🔍 Задание 1: Проверка отображения созданного заказа
* **Цель:** Проверить корректность связи таблиц и отображения активных заказов.
* **Логика:** Вывести список логинов курьеров с количеством их заказов в статусе «В доставке» (`inDelivery = true`).
* **Решение:** Написан SQL-запрос с использованием оператора `JOIN` и агрегатной функции `COUNT`.

> ⚠️ **Обнаруженный дефект (Баг-репорт):** В ходе выполнения задания был зафиксирован UI/Бэкенд баг. При принятии заказа в мобильном приложении курьера, во вкладке "Мои" заказ визуально дублируется. SQL-запрос в базу данных подтвердил дефект: в поле `orders_count` вместо ожидаемой `1` отобразилось значение `2`, что доказывает задвоение связей на уровне БД.

### 📊 Задание 2: Валидация статусов заказов
* **Цель:** Убедиться, что числовые статусы заказов записываются в базу данных корректно согласно правилам бизнес-логики.
* **Логика перевода статусов:**
  * `finished == true` ➡️ Статус **2**
  * `cancelled == true` ➡️ Статус **-1**
  * `inDelivery == true` ➡️ Статус **1**
  * Остальные случаи ➡️ Статус **0**
* **Решение:** Написан SQL-запрос с использованием условной конструкции `CASE WHEN`.
